Formularios web

Todos los formularios en la web, manejan métodos para el envio de las diferentes peticiones que realiza el usuario. Considerando la arquitectura cliente-servidor, el manejo de las peticiones se hace por medio del protocolo http y actualmente por https, cada petición debe ser atendida en un tiempo de respuesta conforme al modelo TCP/IP. Los métodos principales de los formularios son:


    • GET:

      Este método se encarga de enviar la información del formulario por medio de la salida estandar (conocida como URL) del navegador, esto propicia que puedan existir inyecciones de código realizando violaciones a la seguridad del sistema.

      POST:

      Este método se encarga de enviar la información para poder registrar un cambio o un nuevo registro dentro del sistema, enviandola igual por el método estandar pero mediante un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo de la información.

    • En resumen los métodos son:
    • GET: Sirve para enviar y realizar operaciones de lectura o consulta
      POST: Sirve para enviar información de un recurso al servidor con el fin de registrarlo
      DELETE: Sirve para eliminar un recurso especifico dentro del servidor, obteniendo una respuesta de actualización
      PUT: Sirve para almacenar la información de un recurso en particular y conectarse al servicio para realizar la actualización
      HEAD: Sirve para regresar el código de respuesta de cualquier cabecera HTTP asociada a un recurso
